# grammar.md — Лексическая и синтаксическая грамматика

Грамматика описана в форме EBNF.
Обозначения:
- `{ A }` — ноль или более повторений A
- `[ A ]` — A необязательно (ноль или одно)
- `A | B` — A или B
- `"..."` — терминал (литерал)
- `( A B )` — группировка

---

## 1. Лексика

### 1.1 Алфавит

Язык использует кодировку UTF-8. Идентификаторы и ключевые слова состоят из символов ASCII. Строковые литералы могут содержать произвольные UTF-8 символы.

### 1.2 Пробельные символы и комментарии

Пробельные символы (пробел, табуляция, перевод строки, возврат каретки) игнорируются и служат только разделителями токенов.

```
whitespace   ::= ( ' ' | '\t' | '\n' | '\r' ) { ' ' | '\t' | '\n' | '\r' }

line_comment ::= "--" { any_char_except_newline } newline
```

Комментарии не попадают в поток токенов и не влияют на семантику программы.

### 1.3 Ключевые слова

Следующие слова зарезервированы и не могут использоваться как идентификаторы:

```
-- управление, объявления
fn        var       const     return    if        else
while     for       in        break     continue  struct
namespace type      cast      true      false     hollow
impl      self      defer

-- конструкторы типов
array     range

-- имена встроенных типов
int8      int16     int32     int64
uint8     uint16    uint32    uint64
float32   float64
bool      string
```

Имена встроенных типов (`int8..int64`, `uint8..uint64`, `float32`, `float64`, `bool`, `string`, а также уже зарезервированный `hollow`) — **полноценные ключевые слова**, не идентификаторы. Это исключает неоднозначность вида `var int32 = 5;` и согласует подход с `hollow`.

Конструкторы типов `array` и `range` тоже являются ключевыми словами — они появляются только в синтаксисе типа (`array[T, N]`, `range[T]`) и должны однозначно опознаваться лексером.

Встроенные функции (`print`, `println`, `input`, `exit`, `panic`, `len`) **не** являются ключевыми словами — они являются предопределёнными идентификаторами и могут быть перекрыты пользователем.

### 1.4 Атрибуты эффектов

Аннотации эффектов записываются как идентификаторы с префиксом `@`:

```
effect_attr ::= "@" identifier
```

Распознаваемые компилятором атрибуты эффектов: `@pure`, `@io`, `@panics`. Семантика — см. `semantics.md`, §12.

### 1.5 Идентификаторы

```
letter     ::= 'a'..'z' | 'A'..'Z'
digit      ::= '0'..'9'
identifier ::= ( letter | '_' ) { letter | digit | '_' }
```

Идентификатор не должен совпадать ни с одним ключевым словом.

### 1.6 Литералы

#### Целые числа

```
int_literal  ::= digit { digit } [ int_suffix ]
               | "0x" hex_digit { hex_digit } [ int_suffix ]
hex_digit    ::= digit | 'a'..'f' | 'A'..'F'
int_suffix   ::= "u" | "L" | "uL" | "Lu"
```

#### Вещественные числа

```
float_literal ::= digit { digit } '.' digit { digit } [ exponent ] [ float_suffix ]
                | digit { digit } exponent [ float_suffix ]
exponent      ::= ( 'e' | 'E' ) [ '+' | '-' ] digit { digit }
float_suffix  ::= "f"
```

#### Булевы значения

```
bool_literal ::= "true" | "false"
```

#### Строки

```
string_literal ::= '"' { string_char } '"'
string_char    ::= any_utf8_char_except_'"'_and_'\\'
                 | '\\' escape_seq
escape_seq     ::= '"' | '\\' | 'n' | 't' | 'r' | '0'
```

### 1.7 Операторы и разделители

```
+   -   *   /   %          -- арифметические
==  !=  <   >   <=  >=     -- сравнение
&&  ||  !                  -- логические
|>                         -- pipe (композиция)
..                         -- диапазон (полуинтервал)
=                          -- присваивание
->                         -- тип возврата функции
::                         -- доступ к пространству имён / методу типа
.                          -- доступ к полю или методу
,                          -- разделитель параметров/элементов
:                          -- разделитель имя:тип
;                          -- конец инструкции
@                          -- префикс атрибута эффекта
(  )                       -- группировка, вызов функции
{  }                       -- блок
[  ]                       -- индексирование, тип массива
<  >                       -- параметр типа в cast
```

## 2. Синтаксис

### 2.1 Структура программы

```ebnf
program ::= { top_level_decl }
```

На верхнем уровне допускаются только объявления. Инструкции и выражения, не являющиеся частью объявления, допускаются исключительно внутри тела функций.

```ebnf
top_level_decl ::= fn_decl
                 | struct_decl
                 | impl_block
                 | namespace_decl
                 | type_alias_decl
                 | var_decl
                 | const_decl
```

### 2.2 Объявления

#### Переменная

Тип может быть указан явно либо опущен — в этом случае он выводится из инициализатора (см. `types.md`, §5 — вывод типов).

```ebnf
var_decl   ::= "var"   identifier [ ":" type ] "=" expr ";"
const_decl ::= "const" identifier [ ":" type ] "=" expr ";"
```

Инициализатор обязателен всегда, независимо от наличия аннотации типа.

#### Синоним типа

```ebnf
type_alias_decl ::= "type" identifier "=" type ";"
```

#### Функция

```ebnf
fn_decl ::= "fn" identifier "(" [ param_list ] ")" "->" return_type
            [ effect_list ] block_expr

param_list  ::= param { "," param }
param       ::= identifier ":" type

return_type ::= type

effect_list ::= effect_attr { "," effect_attr }
```

Тело функции синтаксически — это `block_expr` (см. §2.5). Однако его финальное значение **игнорируется**: возврат значения из функции осуществляется **только** через инструкцию `return expr;`. Если блок-тело завершается финальным выражением без `;`, это выражение вычисляется, но его значение отбрасывается, как у любого `expr_stmt`.

То же справедливо для тел методов и ассоциированных функций в блоках `impl` (правило `method_decl` ниже).

Достижение конца тела функции с не-`hollow` типом возврата без явного `return` — ошибка компиляции (см. `semantics.md`, §7.7).

Аннотации эффектов необязательны. Семантика эффектов описана в `semantics.md`, §12.

Функции могут быть **перегружены**: в одной области видимости допускается несколько объявлений функции с одним именем, если они различаются количеством или типами параметров. Правила разрешения перегрузок — см. `semantics.md`, §13.

Точка входа — функция `main` с типом возврата `int32`:

```
fn main() -> int32 { ... }
```

#### Структура

```ebnf
struct_decl ::= "struct" identifier "{" field_list "}"

field_list  ::= field { "," field } [ "," ]
field       ::= identifier ":" type
```

#### Блок реализации методов

Методы структуры объявляются в отдельном блоке `impl`:

```ebnf
impl_block ::= "impl" identifier "{" { method_decl } "}"

method_decl ::= "fn" identifier "(" [ method_param_list ] ")" "->" return_type
                [ effect_list ] block_expr

method_param_list ::= self_param [ "," param_list ]
                    | param_list

self_param ::= "self" ":" identifier
```

Первый параметр метода, если он присутствует как `self: T`, является неявным получателем вызова через точку. Имя типа в `self: T` должно совпадать с именем структуры блока `impl`.

Метод без `self` (ассоциированная функция) вызывается через `::`: `StructName::method(...)`.

Для одной структуры допускается несколько блоков `impl` — они объединяются семантическим анализатором.

#### Пространство имён

```ebnf
namespace_decl ::= "namespace" identifier "{" { top_level_decl } "}"
```

### 2.3 Типы

```ebnf
type ::= builtin_type
       | array_type
       | range_type
       | identifier                    -- пользовательский тип (struct или alias)
       | identifier "::" identifier    -- тип из пространства имён

builtin_type ::= "int8"  | "int16"  | "int32"  | "int64"
               | "uint8" | "uint16" | "uint32" | "uint64"
               | "float32" | "float64"
               | "bool"
               | "string"
               | "hollow"

array_type ::= "array" "[" type "," int_literal "]"
range_type ::= "range" "[" type "]"
```

Тип `range[T]` — это ленивая последовательность значений типа `T`, порождаемая выражением диапазона `a..b` (см. §2.5). Используется только в заголовке `for`-цикла и как тип параметра функции. Переменные типа `range[T]` могут быть объявлены, но не могут быть скопированы (см. `semantics.md`, §7.5).

### 2.4 Инструкции

```ebnf
stmt ::= var_decl
       | const_decl
       | assign_stmt
       | expr_stmt
       | if_stmt
       | while_stmt
       | for_stmt
       | return_stmt
       | break_stmt
       | continue_stmt
       | defer_stmt
       | block_stmt
       | empty_stmt

assign_stmt ::= lvalue "=" expr ";"
lvalue      ::= identifier
              | postfix_expr "[" expr "]"
              | postfix_expr "." identifier

expr_stmt   ::= expr ";"

if_stmt     ::= "if" expr block_expr [ "else" ( if_stmt | block_expr ) ]

-- Правило различения if_stmt / if_expr (см. §2.5):
-- Различение происходит **по позиции вызывающего правила**, не по содержимому
-- веток. Конкретно:
--   * Если `if` парсится из правила `stmt` (то есть стоит в позиции инструкции),
--     это всегда `if_stmt`. Ветка `else` необязательна; значение if игнорируется.
--   * Если `if` парсится из правила `primary_expr` (то есть стоит в позиции
--     выражения — в правой части `var`/`const`/`=`, как аргумент, как операнд
--     и т.п.), это всегда `if_expr`. Ветка `else` обязательна (см. §2.5);
--     типы веток должны совпадать.
-- Это решение не требует backtracking и реализуется напрямую методом
-- рекурсивного спуска: правило, вызывающее парсинг, заранее знает позицию.

while_stmt  ::= "while" expr block_expr

for_stmt    ::= "for" identifier "in" expr block_expr

return_stmt ::= "return" [ expr ] ";"

break_stmt    ::= "break" ";"
continue_stmt ::= "continue" ";"

defer_stmt    ::= "defer" stmt

block_stmt    ::= block_expr

empty_stmt    ::= ";"
```

`else if` записывается естественно:

```
if expr { ... } else if expr { ... } else { ... }
```

Грамматически это раскрывается как `else` + новый `if_stmt`.

#### Цикл `for`

Синтаксис `for IDENT in EXPR BLOCK` работает в двух режимах, определяемых типом `EXPR`:

- `EXPR` имеет тип `range[T]` — `IDENT` последовательно принимает значения из диапазона.
- `EXPR` имеет тип `array[T, N]` — `IDENT` последовательно принимает значения элементов массива.

Тип переменной цикла выводится из типа итерируемого выражения. `IDENT` иммутабельна в теле цикла.

#### `defer`

`defer stmt` регистрирует инструкцию к выполнению при выходе из содержащего её блока — как естественном, так и через `return`, `break`, `continue`. Несколько `defer` в одном блоке выполняются в порядке, обратном регистрации (LIFO). Семантика — см. `semantics.md`, §7.8.

### 2.5 Выражения

Блоки в этом языке являются выражениями. Большинство конструкций, которые в C-подобных языках считаются инструкциями (`if`, блок `{ ... }`), имеют тип и значение.

```ebnf
expr ::= pipe_expr

pipe_expr  ::= or_expr { "|>" or_expr }
or_expr    ::= and_expr { "||" and_expr }
and_expr   ::= eq_expr  { "&&" eq_expr  }
eq_expr    ::= cmp_expr { ( "==" | "!=" ) cmp_expr }
cmp_expr   ::= range_expr { ( "<" | ">" | "<=" | ">=" ) range_expr }
range_expr ::= add_expr [ ".." add_expr ]
add_expr   ::= mul_expr { ( "+" | "-" ) mul_expr }
mul_expr   ::= unary_expr { ( "*" | "/" | "%" ) unary_expr }

unary_expr ::= "-" unary_expr
             | "!" unary_expr
             | postfix_expr

postfix_expr ::= primary_expr { postfix_op }
postfix_op   ::= "[" expr "]"          -- индексирование массива
               | "." identifier        -- доступ к полю
               | "." identifier "(" [ arg_list ] ")"  -- вызов метода
               | "(" [ arg_list ] ")"  -- вызов функции

arg_list ::= expr { "," expr }

primary_expr ::= int_literal
               | float_literal
               | bool_literal
               | string_literal
               | array_literal
               | struct_literal
               | cast_expr
               | if_expr
               | block_expr
               | identifier
               | "self"
               | namespace_access
               | "(" expr ")"

namespace_access ::= identifier "::" identifier

cast_expr ::= "cast" "<" type ">" "(" expr ")"

-- Правило для парсера:
-- После ключевого слова "cast" следующий токен `<` всегда трактуется как
-- начало параметра типа, а парный к нему `>` — как закрытие параметра типа,
-- а не операторы сравнения. Внутри параметра типа выражения (а значит, и
-- сравнения) не допускаются — грамматика требует именно `type`. Это снимает
-- неоднозначность токенов `<`, `>` без backtracking: парсер переключает режим
-- лексической интерпретации этих токенов на основании контекста (после `cast`).
-- Аналогичный приём используется в C# для родового синтаксиса.

array_literal  ::= "[" expr { "," expr } "]"

-- Пустой `[]` синтаксически недопустим: размер массива является частью типа
-- (`array[T, N]` с `N > 0`, см. types.md §2.6), и для пустого литерала нельзя
-- ни вывести тип элемента, ни сослаться на корректный тип `array[T, 0]`.
-- Такое требование грамматики согласует синтаксис с системой типов.
struct_literal ::= identifier "{" [ field_init { "," field_init } ] "}"
field_init     ::= identifier ":" expr

if_expr    ::= "if" expr block_expr "else" ( if_expr | block_expr )

block_expr ::= "{" { stmt } [ expr ] "}"
```

#### Значение блока

Блок `block_expr` — это последовательность инструкций, опционально завершающаяся выражением **без `;`**. Значение блока:

- если блок завершается выражением без `;` — значение этого выражения;
- иначе — тип `hollow` и значение отсутствует.

Пример:
```
var x: int32 = {
    var a: int32 = 10;
    var b: int32 = 20;
    a + b                 -- без ";" — это значение блока
};                        -- x == 30
```

#### `if` как выражение

`if_expr` — форма `if`, используемая в позиции выражения. Отличается от `if_stmt` тем, что:

- ветка `else` **обязательна**;
- обе ветки должны иметь совместимый тип (см. `types.md`).

```
var m: int32 = if a > b { a } else { b };
```

Синтаксически парсер различает `if_stmt` и `if_expr` **по позиции вызывающего правила**, не по содержимому тел веток:

- Парсинг из правила `stmt` (см. §2.4) → всегда `if_stmt`. Ветка `else` необязательна, значение `if` игнорируется. Тип такого `if` — `hollow`.
- Парсинг из правила `primary_expr` (правая часть `var`/`const`/`=`, аргумент вызова, операнд бинарного выражения, тело блока в качестве финального выражения и т.п.) → всегда `if_expr`. Ветка `else` обязательна; обе ветки должны иметь совместимый тип.

Это правило однозначно и реализуется без backtracking методом рекурсивного спуска. Семантика обоих узлов идентична с точки зрения генерации кода (см. `codegen.md`, §6.8); различие — только в требованиях к ветке `else` и согласованности типов.

#### Оператор `|>` (pipe)

Выражение `a |> f` семантически эквивалентно вызову `f(a)`. Правая часть pipe должна быть вызываемой: обычный идентификатор функции, доступ в пространство имён, либо частичное применение вида `f(b, c)` — в этом случае `a |> f(b, c)` раскрывается как `f(a, b, c)` (то есть `a` подставляется **первым** аргументом).

Оператор `|>` левоассоциативен:
```
a |> f |> g  ≡  g(f(a))
```

Правая часть pipe не может быть произвольным выражением — только вызовом функции или именем функции. Несоответствие — синтаксическая ошибка.

#### Оператор `..` (диапазон)

Выражение `a..b` порождает значение типа `range[T]`, где `T` — целочисленный тип обоих операндов. Это **полуинтервал**: включает `a`, не включает `b`. Оба операнда должны быть одного целочисленного типа; `b < a` даёт пустой диапазон.

Выражение `..` имеет собственный уровень приоритета между сравнением и арифметикой (см. §2.6) и **не ассоциативно**: `a..b..c` — синтаксическая ошибка.

#### Вызов метода

`obj.method(args)` — вызов метода `method` типа `obj`. Метод должен быть определён в одном из `impl`-блоков для типа `obj`. Разрешение происходит в семантическом анализаторе.

#### `self` как первичное выражение

Ключевое слово `self` допустимо только внутри тела метода и обозначает получателя вызова.

### 2.6 Приоритет операторов

Приоритет от наименьшего к наибольшему:

| Уровень | Операторы               | Ассоциативность |
|---------|-------------------------|-----------------|
| 1       | `\|>`                   | левая           |
| 2       | `\|\|`                  | левая           |
| 3       | `&&`                    | левая           |
| 4       | `==` `!=`               | левая           |
| 5       | `<` `>` `<=` `>=`       | левая           |
| 6       | `..`                    | не ассоциативен |
| 7       | `+` `-`                 | левая           |
| 8       | `*` `/` `%`             | левая           |
| 9       | унарный `-`, `!`        | правая          |
| 10      | `[]` `.` `()`           | левая           |

### 2.7 Порядок вычисления операндов

Операнды бинарных операторов вычисляются слева направо. Аргументы вызова функции вычисляются слева направо. Для `a |> f(args)` сначала вычисляется `a`, затем `args` слева направо.

## 3. Встроенные функции

Встроенные функции являются предопределёнными идентификаторами и вызываются как обычные функции:

```
print(expr)      -- вывод значения в stdout БЕЗ перевода строки; возвращает hollow
println(expr)    -- вывод значения в stdout С переводом строки; возвращает hollow
println()        -- вывод только перевода строки; возвращает hollow
input()          -- чтение строки из stdin; возвращает string; @io
exit(expr)       -- завершение программы с кодом возврата int32; не возвращает
panic(expr)      -- аварийное завершение с сообщением string; @panics
len(s)           -- длина строки в байтах; возвращает int32; @pure
```

Эффекты встроенных функций учитываются при проверке аннотаций `@pure` пользовательских функций (см. `semantics.md`, §12).

## 4. Примеры

### Hello, world

```
fn main() -> int32 {
    println("Hello, world!");
    return 0;
}
```

### Вывод типов и выражение-блок

```
fn main() -> int32 {
    var a = 10;                      -- int32 выведен из литерала
    var b = 3.14;                    -- float64 выведен из литерала
    const name = "Kostya";           -- string

    var max = if a > 5 { a } else { 0 };
    println(max);
    return 0;
}
```

### Методы структур

```
struct Point {
    x: float64,
    y: float64,
}

impl Point {
    fn make(x: float64, y: float64) -> Point @pure {
        return Point{ x: x, y: y };
    }

    fn distance(self: Point, other: Point) -> float64 @pure {
        var dx = self.x - other.x;
        var dy = self.y - other.y;
        return sqrt_approx(dx * dx + dy * dy);
    }
}

fn sqrt_approx(v: float64) -> float64 @pure {
    -- ... упрощённо
    return v;
}

fn main() -> int32 {
    var p = Point::make(3.0, 4.0);
    var q = Point::make(0.0, 0.0);
    println(p.distance(q));
    return 0;
}
```

### Перегрузка функций

### Pipe-оператор

```
fn double(x: int32) -> int32 @pure { return x * 2; }
fn inc(x: int32)    -> int32 @pure { return x + 1; }

fn main() -> int32 {
    var result = 5 |> double |> inc;   -- inc(double(5)) == 11
    println(result);
    return 0;
}
```
